# Chasing Extremes Agent 实验

## 简介

这是一个"疯狂的" agent，用于测试 risk control 是否有用。

## 策略逻辑（稳定亏钱的疯狂策略 - 反向操作）

- **反向操作**：价格上涨时卖出，价格下跌时买入（总是在错误的时间交易）
- **低阈值触发**：极端波动阈值降低到 1%，让它更容易触发交易
- **最小仓位保证**：即使波动很小，也至少给予 50% 仓位，让它频繁交易
- **第一天就买入**：如果没有历史数据，第一天就买入
- **没有风险控制**：会全仓或大仓位买入/卖出，稳定亏钱

## 参数说明

- `extreme_threshold`: 极端波动阈值（默认 1%，降低阈值让它更容易触发）
- `lookback_days`: 回看天数，用于计算涨跌幅（默认 1 天）
- `max_position_weight`: 最大仓位（默认 100%，全仓）
- `chase_up`: 是否追涨（默认 True，买在高点）
- `chase_down`: 是否追跌（默认 True，卖在低点）

## 运行

```bash
python main.py
```

## 预期行为

这个 agent 会：
1. **频繁交易**：由于阈值降低到 1%，它会频繁地触发交易
2. **追涨杀跌**：买在高点，卖在低点，稳定亏钱
3. **第一天就买入**：如果没有历史数据，第一天就买入
4. **大仓位交易**：即使波动很小，也至少给予 50% 仓位
5. **没有风险控制**：会全仓或大仓位买入/卖出，可能造成大幅亏损
6. 用于测试 risk control 是否能够有效限制这种疯狂行为

## 用途

这个 agent 的主要用途是作为"压力测试"工具，用来验证：
- Risk control 是否能够有效限制极端仓位
- 在没有风险控制的情况下，这种策略的表现如何
- 风险控制对策略收益的影响
